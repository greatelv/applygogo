# Boost Traffic Skill (트래픽 부스팅 스킬)

이 스킬은 `applygogo.com`의 검색 엔진 노출(SEO)을 극대화하고, 방문자의 이탈을 막아 서비스 이용으로 전환시키기 위한 **종합 진단 및 최적화 가이드**입니다.

## 1. 수행 목표

- 검색 엔진(Google, Naver 등)이 페이지를 더 잘 이해하고 상위에 노출하도록 기술적/콘텐츠적 요소를 최적화합니다.
- 소셜 미디어 공유 시 클릭률(CTR)을 높이기 위한 메타 데이터를 점검합니다.
- 다국어 사용자가 올바른 언어 페이지로 유입되도록 설정을 검증합니다.

## 2. 체크리스트 및 분석 항목

### A. 메타 데이터 및 SEO (Technical SEO)

1. **Title & Description 최적화**:
   - 각 페이지(특히 랜딩 페이지)의 `title`이 매력적이고 키워드를 포함하고 있는가?
   - `meta description`이 클릭을 유도하는 문구이며 적절한 길이를 유지하고 있는가?
2. **Open Graph (OG) 태그**:
   - `og:image`, `og:title`, `og:description`이 설정되어 있으며, 공유 시 미리보기가 매력적인가?
   - 동적 라우트나 다국어 페이지에 맞는 OG 이미지가 적용되었는가?
3. **Canonical & Alternate URL**:
   - 페이지의 대표 주소(`canonical`)가 명시되어 있는가?
   - 다국어 지원을 위한 `hreflang` 태그(x-default 포함)가 타겟 언어별로 올바르게 설정되었는가?

### B. 콘텐츠 및 키워드 (On-Page SEO)

1. **헤딩 태그 구조 (H1~H6)**:
   - 페이지 당 하나의 `<h1>` 태그가 핵심 키워드를 포함하고 있는가?
   - `<h2>`, `<h3>` 등으로 콘텐츠의 구조가 논리적으로 잡혀있는가?
2. **키워드 배치**:
   - 타겟 키워드(예: "이력서 번역", "AI Resume", "Resume Translator")가 자연스럽게 본문에 녹아있는가?

### C. 파일 및 성능 (Performance for SEO)

1. **robots.txt & sitemap.xml**:
   - 크롤러의 접근을 막고 있지는 않은가?
   - 사이트맵이 최신 경로를 포함하고 있는가?
2. **이미지 최적화**:
   - 주요 이미지에 `alt` 태그가 설명적으로 작성되어 있는가?

## 3. 실행 가이드 (Process)

사용자가 "유입을 늘려줘", "SEO 점검해줘", "홍보가 잘 되게 해줘" 등을 요청하면 다음 순서로 작업을 진행하세요:

1. **현재 상태 분석 (`analyze mode`)**:
   - 주요 랜딩 페이지(`src/app/page.tsx`, `src/app/[locale]/page.tsx`)와 `layout.tsx`를 `view_file`로 읽습니다.
   - `sitemap.xml`, `robots.txt` 파일의 존재 여부와 내용을 확인합니다.
   - 위 체크리스트를 기반으로 현재 코드의 문제점을 리포트 형식으로 작성합니다.

2. **개선 제안 (`propose mode`)**:
   - 발견된 문제점을 해결하기 위한 구체적인 코드 수정안을 제시합니다.
   - 예: "현재 메타 설명이 비어있습니다. 'AI로 3초 만에 완성하는 글로벌 이력서'라는 문구를 추가하는 것을 추천합니다."

3. **적용 (`apply mode`)**:
   - 사용자의 승인이 떨어지면, 실제로 코드를 수정하여 최적화를 적용합니다.

## 4. 참고 사항 (ApplyGoGo 특화)

- **다국어 특성**: ApplyGoGo는 한국어, 영어, 일본어 등을 지원하므로, SEO 작업 시 항상 **다국어 메타 데이터 동기화**를 최우선으로 고려해야 합니다.
- **최신 기술 스택**: Next.js App Router를 사용 중이므로 `generateMetadata` 함수를 활용한 동적 메타 데이터 생성을 적극 활용하세요.
