# 핵심 행동 수칙 (CRITICAL RULES)

1. **질문 및 제안에 대한 선(先)실행 절대 금지**
   - 사용자가 "이거 하면 어때?", "가능해?", "왜 안 돼?" 등의 **질문이나 의견**을 제시했을 때, **절대 코드를 먼저 수정하지 마십시오.**
   - 반드시 **1) 원인 분석 및 답변 제공 -> 2) 수정 제안 -> 3) 승인 대기** 프로세스를 따르십시오.
   - 코드를 수정하려면 사용자의 명시적인 지시("수정해줘", "적용해")가 있어야 합니다.

2. **모든 설명과 구현계획은 한국어로 응답**

3. **브라우저 자동화 효율성 및 정확성 극대화**
   - 브라우저 작업 시 불필요한 대기(Wait)를 최소화하고, 가능한 한 JavaScript 실행(`execute_browser_javascript`)을 사용하여 신속하고 정확하게 작업을 수행하십시오.
   - 입력 필드(Email, Password 등)에 값을 입력할 때는 기존 값이 남아있지 않도록 반드시 필드를 비우거나, JavaScript로 `value`를 직접 설정하고 적절한 이벤트(`input`, `change`)를 발생시켜 데이터 혼선(예: 이메일 중복 입력)을 방지하십시오.
   - 타임아웃이나 대기 조치는 시스템 반응 속도에 맞춰 가장 효율적인 최소한의 시간만 사용하십시오.

4. **글로벌 로케일(Locale) 및 다국어 처리 철저 준수**
   - 모든 UI 텍스트, 라벨, 메시지 작성 시 반드시 다국어(ko, en, ja) 처리를 고려하십시오.
   - 하드코딩된 텍스트 대신 `messages/*.json` 파일에 번역 키를 추가하여 사용하십시오.
   - 한국어 페이지(`locale === "ko"`)에서는 브랜드명으로 **'지원고고'**를 사용하고, 그 외 글로벌 페이지에서는 **'ApplyGogo'**를 사용하십시오.
   - 데이터 파일(JSON 등) 구조 설계 시에도 다국어 지원을 위한 로케일 객체 구조를 기본으로 채택하십시오.
   - **릴리즈 노트 관리**: 릴리즈 노트는 `main` 브랜치 푸시 시 AI(`auto-generate-release-notes.ts`)에 의해 자동으로 생성됩니다. 수동 추가가 필요할 경우 반드시 다국어 입력을 지원하는 스크립트(`scripts/add-release-note.ts`)를 사용하십시오.
